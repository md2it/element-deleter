# SPECIFICATION RU: ELEMENT DELETER

---

## ОПИСАНИЕ

- Браузерное расширение
- Удаление/восстановление элементов веб страницы

[README-RU.md] даёт более детальное описание "для третьих лиц":
- Для пользователей: сравнение с аналогами, пользовательские пути, ограничения и т.п.
- Для сторонних разработчиков: рекомендации по сборке

## ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

### Включение и выключение
**Запуск:**
- По нажатию на иконку расширения в панели браузера
- По хоткею (может быть выключено)
**Завершение:**
- По повторному нажатию на иконку расширения в панели браузера
- По хоткею (два варианта хоткеев, они могут быть выключены)
- По кнопке в уведомлении
- При открытии окна расширения (popup)

### Выделение
- Выделяется самый дочерний DOM узел под курсором. Даже если материнский элемент прозрачный и того же размера
- Ограничения:
   - Выделение не меняет вёрстку
   - Не выделяются элементы нашего расширения
- Рамка выделения повторяет форму элемента, включая сложные формы
- Информация о теге
   - Отключаемая опция
   - Левый нижний угол подписи совпадает с левым верхним углом рамки выделения
   - Выводит по первому совпадению:
      1. `tag#id:pseudo-class` если есть ID и pseudo-class
      2. `tag#id` если есть ID
      3. `tag.class:pseudo-class` если есть class (все классы) и pseudo-class
      4. `tag.class` если есть class (все классы)
      5. `tag`
- iFrame отдельный обработчик

### Удаление
- Удаляет выбранный элемент DOM вместе с дочерними элеменатми
- iFrame удаляется целиком, т.к. ограничение платформы
- Вызывает уведомление
- Многократное удаление:
   - После удаления элемента U может указать и удалить другой элемент
   - Аналогично работают: подсветка, удаление, уведомление

### Восстановление элемента
- По кнопке в уведомлении:
   - Восстанавливает соответствующий элемент
- По хоткею:
   - Восстанавливает последний удалённый элемент
   - Многократное использование хоткея восстанавливает элементы в обратном порядке от порядка удаления
- Вызывает уведомление о восстановлении
- Бережём память клиента: 
   - "Запоминание" работает только в рамках текущей страницы
   - Закрытие или перезагрузка страницы очищают память

### Открытие окна (popup)
- По кнопке в уведомлении
- Из контекстного меню расширения

### Хоткей префикс-режим
- Вкл/выкл:
   - Default `Ctrl+Shift+X` → `D`
   - Mac `Cmd+Shift+X` → `D`

### Хоткей простой
- `Esc` — выкл режима удаления
- `Ctrl+Z` / `⌘+Z` — восстановить

## UX UI ТРЕБОВАНИЯ

### Ключевой цвет приложения
- #b91c1c

### Иконка расширения
- Считается активной если:
   - Готово выделять элементы под курсором
   - Открыт popup расширения (окно настроек / о расширении)
- Иконка всегда trash #fff
- Фон в зависимости от состояния расширения:
   - #012292: Неактивно
   - #b91c1c: Активно

### Page context menu
- Кнопка "удалить этот элемент":
   - Удаляет именно тот элемент, на котором был курсор в момент вызова данного контекстного меню

### Уведомления

**Об удалении**
- Красное
- Содержит в ряд:
   - Иконку расширения
   - Текст "DELETED: <дескриптор>" (тот же формат, что у подписи при выделении)
      - Если дескриптор слишком длинный, обрезается
   - Кнопка восстановления (иконка undo-2)
   - Кнопка настроек (иконка cog)
   - Кнопка инфо (иконка info)

**О восстановлении**
- Синее
- Наполнение:
   - Аналогично "DELETED"
   - Но "DELETED" -> "RESTORED", нет кнопки восстановления

### Окно
- Header:
   - Слева логотип: иконка расширения, высота как у блока "название + пояснение"
   - По центру друг над другом: заголовок "ELEMENT DELETER" и пояснение "браузерное расширение"
- Переключатель табов (SETTINGS | ABOUT)
- Красная разделительная линия (#b91c1c)
- Контент (зависит от таба)
- Красная разделительная линия (#b91c1c)
- Footer со ссылками

**SETTINGS**
- Настройка языка:
   - Язык
   - Простые кнопки: EN | ES | FR | DE | RU | 中文 | عربي
   - Блок контента: именно этот список не меняет ориентацию при смене LTR / RTL
- Настройка времени уведомлений:
   - Поле со стрелочками (шевронами)
   - Только целочисленные значения от 0 до 10
   - Растянуто по ширине
   - В центре текст "Уведомления N сек"
   - N это редактируемое число
   - Редактирование должно быть возможно при клике на "Уведомления N сек"
   - Тултип "Установите 0 для выключения уведомлений"
- Toggler: "Вкл/выкл `Ctrl+Shift+X` → `D`" или "Вкл/выкл `Cmd+Shift+X` → `D`" (стрелка вне бэктиков, пробелы вокруг →; строка зависит от ОС)
- Toggler: "Выкл `Esc`"
- Toggler: "Отменить удаление `Ctrl + Z`", "Отменить удаление `Ctrl + Z`" (строка и настройка зависит от ОС)
- Toggler: "`tag#id` / `tag.class`"
- По умолчанию:
   - Язык: пытается угадать настройки пользователя (fallback EN)
   - Время уведомления = 4
   - Все хоткеи включены
   - "tag#id / tag.class" выключено

**ABOUT**
- Boolean список:
   - <trash-2> Удаляет элемент страницы,
   - <circle-power> Вкл/выкл в один клик,
   - <undo-2> Можно восстановить элемент,
   - <undo-2> Перезагрузка страницы восстановит всё,
   - <shield-check> Не использует сеть,
   - <shield-check> Не собирает данные,
   - <синяя иконка как в панели браузера> Расширение выключено,
   - <красная иконка как в панели браузера> Расширение запущено.

### Анимации
- Рамка выделения анимированная, чтобы человек лучше видел
- Плавный переход между выделенными элементами
- Окно расширения (popup) появляется и пропадает плавно
- Перед полным удалением элемента: становится мелким и прозрачным с красноватым оттенком
- Восстановление обратное, но с синим оттенком (как тост «Восстановлено»)
